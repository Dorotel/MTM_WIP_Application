🎯 Development Tools Overview

MTM Inventory Application includes specialized development tools accessible through the MainForm Development menu. These tools are designed for developers, system administrators, and advanced users who need to analyze application structure and dependencies.

📊 Dependency Chart Converter

Converts markdown dependency charts to interactive HTML format

🔍 Dependency Chart Viewer

Views and navigates through generated HTML dependency charts

📊 Dependency Chart Converter

🚀 Accessing the Tool

  1. Open MTM Inventory Application
  2. Click on the Development menu in the menu bar
  3. Select "Dependency Chart Converter"
  4. The converter window will open

⚙️ Tool Interface

Main Controls:

  • Base Path Selection: Choose the root directory containing Documentation folder
  • Convert Charts Button: Process all markdown files to HTML
  • Progress Indicator: Shows conversion progress
  • Status Display: Reports current operation status

📁 File Structure Requirements

The converter expects the following directory structure:

Selected Base Path/
├── Documentation/
│   └── Dependency Charts/
│       ├── Templates/
│       │   └── chart-template.html
│       ├── Forms/
│       ├── Controls/
│       ├── Services/
│       ├── Helpers/
│       └── HTML/ (output directory)

🔄 Conversion Process

  1. Select Base Path: Click "Select Base Path" and choose the project root directory
  2. Validation: Tool automatically validates directory structure and template presence
  3. Start Conversion: Click "Convert Charts" to begin processing
  4. Progress Monitoring: Watch progress bar and status messages
  5. Completion: HTML files will be generated in the HTML output directory

⚠️ Requirements:

  • ✅ chart-template.html must exist in Templates folder
  • ✅ Valid markdown dependency chart files (*.md)
  • ✅ Write permissions to HTML output directory
  • ✅ Proper file structure as shown above

📝 Supported Markdown Features

The converter processes these markdown elements:

  • ✅ File Headers: # Dependency Chart: FileName.cs
  • ✅ File Information: Path, Type, Complexity, Priority
  • ✅ Dependencies: Internal and External dependency lists
  • ✅ Compliance Status: Region organization compliance
  • ✅ Refactoring Actions: Recommended improvements
  • ✅ Code Snippets: Example code blocks

🔍 Dependency Chart Viewer

🚀 Accessing the Tool

  1. Open MTM Inventory Application
  2. Click on the Development menu in the menu bar
  3. Select "Dependency Chart Viewer"
  4. The viewer window will open

🎮 Viewer Interface

Main Components:

  • HTML Directory Selection: Choose folder containing HTML dependency charts
  • File List: Shows all available HTML files for viewing
  • Web Browser Panel: Displays selected dependency chart
  • Navigation Controls: Forward, back, refresh, and home buttons
  • Status Bar: Shows current file and load status

🔄 Using the Viewer

  1. Select HTML Directory: Click "Select HTML Directory" button
  2. Choose Directory: Navigate to Documentation/Dependency Charts/HTML/
  3. Load Files: Tool automatically scans for HTML files
  4. Select File: Click on any file in the list to view it
  5. Navigate: Use browser controls to explore the dependency chart

🎯 Navigation Features:

  • 🔙 Back Button: Return to previous chart
  • 🔜 Forward Button: Go to next chart
  • 🔄 Refresh Button: Reload current chart
  • 🏠 Home Button: Return to first/default chart
  • ⌨️ Keyboard Shortcuts: F5 (refresh), Alt+Left (back), Alt+Right (forward)

📊 Chart Information Display

Each HTML dependency chart contains:

  • 📁 File Information: Path, type, complexity rating
  • 🔗 Dependency Lists: Internal and external dependencies with status
  • 📊 Compliance Report: Region organization and method ordering status
  • 🔧 Refactoring Actions: Recommended improvements and fixes
  • 💡 Code Examples: Sample code snippets and patterns
  • 🎯 Priority Indicators: Visual priority levels for improvements

🚨 Troubleshooting

❌ Common Issues

🚫 "Template file not found" Error

Cause: chart-template.html is missing from Templates folder

Solution:

  1. Verify Templates folder exists in Documentation/Dependency Charts/
  2. Ensure chart-template.html file is present
  3. Check file permissions for read access

📁 "Directory structure invalid" Error

Cause: Required directory structure is not found

Solution:

  1. Select the correct base path (project root)
  2. Ensure Documentation/Dependency Charts/ folders exist
  3. Create missing directories if necessary

🌐 Browser not loading charts

Cause: Browser control or file access issues

Solution:

  1. Ensure HTML files were successfully generated
  2. Check Windows Defender or antivirus isn't blocking files
  3. Try refreshing the browser with F5
  4. Restart the viewer tool if necessary

🔒 Permission denied errors

Cause: Insufficient file system permissions

Solution:

  1. Run application as administrator if necessary
  2. Check folder permissions for read/write access
  3. Ensure files aren't locked by other applications
