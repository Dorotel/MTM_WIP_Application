When genrating a commit message, the following rules apply:
Commit must me in .md fomat and follow these rules:
// 1. The first line must be a short summary of the changes, no more than 50 characters.
// 2. The first line must be followed by a blank line.
// 3. The second line must be a list of files changed, each on a new line.
// 4. The third line must be a list of files added, each on a new line.
// 5. The fourth line must be a list of files deleted, each on a new line.
// 6. For each file, the format is:
//    <file path> (<status>)
//    a. Paragraph Plain-English description of the change.
//    b. The status can be one of: "modified", "added", "deleted", "renamed", "copied".
//    c.  The file path should be relative to the repository root.
//    d.  The file path should not include the repository name or any leading slashes.
//    e.  The next line after the paragraph description should be blank.
//    f.  The next should be a detailed list of changes made to the file.
//    g.  the detailed list should be in bullet point format, with each bullet point starting with a hyphen and a space.
// 7. The commit message must end with a blank line.
// 8. End with the date, time and timezone in the format "YYYY-MM-DD HH:MM:SS ±HHMM".

// Example commit message:
//
// Fix bug in user authentication
//
// Changed files:
// src/auth.js (modified)
// - Fixed issue with password hashing
// - Improved error handling
// src/user.js (added)
// - Added new user profile feature
// src/oldfile.js (deleted)
// - Removed deprecated code
//
// Date: 2023-10-01 12:00:00 +0000
